### 4. 识别流程
```
启动监控
    ↓
[定时检查 - 每500ms]
    ↓
┌─────────────────────┐
│  检查手牌区是否有牌 │  ← 只在开局时识别一次
└─────────────────────┘
    ↓ 有牌且尚未识别
识别手牌 → 设置game_state.my_hand
    ↓
┌─────────────────────┐
│  检查出牌区是否变化 │
└─────────────────────┘
    ↓ 有新牌
识别出牌
    ↓
判断：是谁出的牌？
    ├─ 上家出牌 → 设置game_state.current_play
    └─ 我出牌   → 追加到game_state.played_cards
                    同时从my_hand中移除
    ↓
更新记牌器
更新出牌建议
更新UI显示
```

## 四、用户使用指南

### 首次使用

1. **运行程序**
   ```bash
   cd d:\Project\ddj_test
   python run.py
   ```

2. **打开微信欢乐斗地主小程序**

3. **校准区域**
   - 点击【校准区域】按钮
   - 点击【截取屏幕】
   - 选择【手牌区】，用鼠标框选手牌区域
   - 选择【出牌区】，用鼠标框选桌面出牌区域
   - 点击【保存配置】

4. **开始监控**
   - 点击【开始监控】
   - 程序会自动识别手牌和出牌

### 游戏中使用

```
开局发牌 → 程序自动识别手牌
    ↓
开始出牌 → 程序自动检测出牌区变化
    ↓
识别到新牌 → 自动判断是谁出的牌
    ↓
    ├─ 上家出牌 → 显示在上家出牌区，生成推荐
    └─ 我出牌   → 从手牌移除，添加到已出牌
```

## 五、技术细节

### 配置文件格式 (config/config.json)
```json
{
  "hand_region": [x, y, w, h],
  "play_region": [x, y, w, h]
}
```

### 变化检测算法
- 使用图像差分比较两帧
- 阈值：手牌区0.15，出牌区0.1
- 连续两帧确认才触发识别

### 智能判断逻辑
- 如果识别的牌都在我的手牌中 → 我出牌
- 否则 → 上家出牌
